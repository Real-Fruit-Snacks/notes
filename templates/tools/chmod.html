Understanding the bits

The three triads

A mode is three groups of three bits — user (the owner), group, and other (everyone else) — each holding read, write and execute. ls -l prints them in exactly that order: -rwxr-xr-x is rwx for the owner, r-x for the group, r-x for the world, and the leading character is the file type (- file, d directory).

Octal: why 755 means rwxr-xr-x

Each triad is one octal digit: read is 4, write is 2, execute is 1, added together. rwx = 4+2+1 = 7; r-x = 4+1 = 5. Three triads, three digits: 755. A fourth, leading digit carries the special bits — setuid 4, setgid 2, sticky 1 — so 4755 is 755 plus setuid.

Symbolic notation

Letters instead of arithmetic: u, g, o pick a class (a or nothing = all), then = sets exactly, + adds, - removes. Clauses chain with commas: u=rwx,go=rx ≡ 755; go-w revokes group/other write and leaves the rest alone — handy when you don't know (or care about) the current mode. A capital X adds execute only where something is already executable.

Directories: x means enter, w means delete

On a directory the bits shift meaning: r lists names, x lets you enter it (and reach anything inside), w creates, renames and deletes entries — which is why deleting a file needs write on the directory, not on the file itself. A directory with r but no x is a list of names you can't touch; 755 on directories is the everyday default.

setuid (4xxx)

An executable with setuid runs as its owner, not as the user who launched it — passwd is root-owned and setuid so ordinary users can update /etc/shadow. In ls -l it replaces the owner's x slot: s if execute is also set (rws), capital S if not — a capital S is almost always a mistake. Most kernels ignore setuid on scripts, and on directories it does nothing on Linux.

setgid (2xxx)

On an executable, setgid runs it with the file's group. The everyday use is on directories: new files created inside inherit the directory's group instead of the creator's primary group — the standard trick for shared project folders (chmod 2775 shared/). Shown in the group's execute slot as s (or S without execute).

Sticky bit (1xxx)

On a world-writable directory, the sticky bit restricts deletion: only a file's owner (or the directory's owner, or root) may remove or rename it. That's what keeps users from deleting each other's files in /tmp, which is mode 1777 — rwxrwxrwt. It sits in the last execute slot as t (or T without execute); its old meaning on executables is long obsolete.

umask: the permission filter

No file is born 777. Programs request a mode — conventionally 666 for files, 777 for directories — and the process's umask clears its bits from the request: with the common 022, files appear as 644 and directories as 755. That subtraction is also why fresh files never have execute. 077 keeps everything private to you; 002 leaves group write for shared work.

Modes you'll meet everywhere

644 ordinary file · 600 private file (SSH keys insist on it) · 755 executable or directory · 700 private directory · 2775 shared group directory · 1777 world-writable with sticky (/tmp) · 400 read-only secret · and 777, which is almost always the wrong answer.

What this calculator assumes

Symbolic = replaces a class's triad and clears that class's special bit unless you list it (u=rwxs keeps setuid) — GNU chmod behaves this way on files, though some systems preserve setuid/setgid on directories unless told explicitly. One operator per comma-clause is supported here; real chmod also allows chains like u+r-w. The umask panel masks the classic 666/777 creation requests.
