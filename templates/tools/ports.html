Understanding ports

What a port actually is

A 16-bit number (0–65535) that lets one IP address host many conversations: a connection is the four-tuple of source IP + source port + destination IP + destination port. The numbers in this table are conventions about destinations — who's expected to be listening — nothing in TCP/IP enforces them.

The three ranges

IANA splits the space in three: well-known (0–1023, the table's bulk, standardized services), registered (1024–49151, first-come registrations from vendors), and dynamic (49152–65535, never assigned — reserved for ephemeral use). Registration is paperwork, not protection: anything can listen anywhere.

TCP, UDP, or both?

IANA registers almost every name for both transports, but reality picks one: SSH is TCP, NTP and DHCP are UDP. The table shows the transport each service actually speaks. The interesting cases genuinely use both — DNS (UDP queries, TCP zone transfers), RDP (UDP for graphics since 8.0), and 443 (TCP for HTTPS, UDP for HTTP/3's QUIC).

Privileged ports

Unix tradition: binding below 1024 requires root, so a connecting client could "trust" the daemon was admin-installed. Containers, CAP_NET_BIND_SERVICE, and Windows (which never had the rule) eroded that story — modern Linux even lets you lower the threshold with a sysctl. Treat it as history, not security.

Ephemeral ports

Your side of a connection gets a temporary source port from the dynamic range — but OSes disagree on its bounds: IANA says 49152–65535, Linux defaults to 32768–60999, and Windows since Vista follows IANA. It matters when you're writing firewall rules for return traffic, or wondering why traceroute probes UDP 33434 and up.

Port 0

Reserved everywhere and unroutable in practice: in the sockets API, binding to 0 means "give me any free port". Packets with source or destination 0 in the wild are almost always crafted — many IDS rules flag them on sight.

Registered ≠ what's running

The registry says 8080 is "http-alt" — in the wild it's Tomcat, proxies, and a million dev servers. 2222 is officially EtherNet/IP-2 but practically alternate SSH; 10000 is NDMP on paper and Webmin in practice. Nothing stops a service from squatting any number, which is why scanners fingerprint banners, not numbers.

The notorious numbers

Some ports are famous for the wrong reasons: 31337 ("elite") was Back Orifice's backdoor, 12345 was NetBus, 4444 is Metasploit's default listener, and 1337 is wherever leetspeak lands. Finding one listening doesn't prove compromise — but it's always worth asking what put it there.

The amplification list

UDP services that answer small questions with big responses get abused for reflected DDoS: chargen (19), DNS (53), NTP (123), SNMP (161), SSDP (1900), and memcached (11211 — 2018's record 1.7 Tbps attack). If you run any of these, rate-limit them, disable the amplifying queries, and never expose them to the open internet.
