Understanding the representations

Character

The glyph itself — or, for characters with no visible shape, the short label the chips above use (SP, TAB, ZWSP, NBSP…). The cell is colored with the same class as its chip.

Code point

Unicode gives every character one number — its code point — written U+ plus at least four hex digits: H is U+0048, 💩 is U+1F4A9. Every other column in this table is derived from that single number.

Decimal

The code point in base 10 — H is 72. This is what JavaScript's codePointAt() returns, and the form used by decimal HTML entities like &#8203;.

Hex

The same number in base 16 — H is 0x48. Hex is the convention throughout Unicode because each hex digit is exactly four bits: byte boundaries stay visible at a glance.

Binary

The raw bits of the code point, padded to whole bytes (8 digits up to U+00FF, 16 through the rest of the Basic Multilingual Plane, 24 above). These are the bits the encodings below repackage.

UTF-8

How the character actually travels over the wire almost everywhere. ASCII stays a single identical byte (0xxxxxxx). Larger code points use 2–4 bytes: a prefix byte announcing the length (110xxxxx, 1110xxxx or 11110xxx) followed by 10xxxxxx continuation bytes, with the code point's bits packed into the x positions — é becomes C3 A9, 💩 becomes F0 9F 92 A9.

UTF-16

What JavaScript (and Windows, and Java) strings are made of. Code points up to U+FFFF are one 16-bit unit. Above that, the code point minus 0x10000 is split across a surrogate pair — D800 plus the high ten bits, DC00 plus the low ten — so 💩 is D83D DCA9, and "💩".length is 2 in JavaScript.

HTML entity

The numeric character reference &#x…; inserts any character into HTML regardless of the page's encoding. A few characters also have memorable named entities — &amp;, &lt;, &nbsp; — but the numeric form works for all of them.

URL-encoded

How the character is written inside a URL: each UTF-8 byte becomes % plus two hex digits (💩 → %F0%9F%92%A9). Unreserved ASCII — letters, digits, - _ . ~ — passes through untouched, which is why h stays h. (This table uses the browser's encodeURIComponent, which also leaves ! ' ( ) * alone.)
