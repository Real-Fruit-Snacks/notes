Understanding the output

CIDR & the prefix

An address block in CIDR notation is the network address plus a prefix length: 192.168.1.0/24. The prefix — /24 here — is how many of the address's 32 bits are locked in as the network part; the remaining 8 bits are free to vary from host to host within the block.

Netmask

The netmask spells the prefix out as four octets: 24 one-bits followed by zero-bits gives 255.255.255.0. Routers AND an address with the mask to recover its network — every address in this block ANDs down to 192.168.1.0.

Wildcard mask

The wildcard mask is the bitwise inverse of the netmask: 0.0.0.255. Cisco ACLs and OSPF use it to say which bits may vary — a 0 bit must match exactly, a 1 bit is free.

Network address

Set every host bit to 0 and you get 192.168.1.0 — the name of the block itself. It identifies the subnet in routing tables and can't be assigned to a machine (/31 and /32 networks are the exception — see Host range).

Broadcast address

Set every host bit to 1 and you get 192.168.1.255. A packet sent there is delivered to every host on the subnet, so it can't be assigned to a single machine either (again, /31 and /32 excepted).

Host range & counts

With 8 host bits the block spans 2^8 = 256 addresses. On ordinary networks two of them — the network and broadcast addresses — are reserved, leaving 254 usable hosts from 192.168.1.1 to 192.168.1.254.

The exceptions: a /31 has no network or broadcast address — both of its two addresses are used by the endpoints of a point-to-point link (RFC 3021) — and a /32 is a route to exactly one host.

Class (A–E)

Before CIDR (1993), the leading bits of an address fixed its class and therefore its size: A (0…, /8), B (10…, /16), C (110…, /24), D (1110, multicast) and E (1111, reserved). This address falls in class C. Classes are purely historical today — with CIDR a network can be any size.

Designation

IANA sets some ranges aside: the RFC 1918 blocks (10/8, 172.16/12, 192.168/16) for private networks, 100.64/10 for carrier-grade NAT, 127/8 for loopback, 169.254/16 for link-local, the TEST-NET blocks for documentation, 224/4 for multicast and 240/4 reserved. This network is Private (RFC 1918) — anything not in a special range is publicly routable.

Reading the binary view

The bit row is the address you typed, one cell per bit. Blue cells are the network portion — the first 24 bits, frozen by the mask; peach cells are the 8 host bits that vary across the block. The purple rule marks the prefix boundary. Read the row with all host bits set to 0 and you get 192.168.1.0; set them all to 1 and you get 192.168.1.255.
